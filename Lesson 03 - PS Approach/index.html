Problem Solving Approach

Problem Solving Approach

When we talk about problem solving approaches we tend to think about algorithms. An algorithm is a process or set of steps to accomplish a certain task. With problem solving in mind we have certain objectives we want to achieve, which are:

  1. Devise a plan to solve algorithms
  2. Compare and contrast problem solving patterns
There are 5 things to keep in mind when devising a plan to solve a problem:
  1. Understand the problem
  2. Explore concrete examples
  3. Break it down
  4. Solve/simplify
  5. Look back and refactor
Let's look at each of these below.

Understand the problem

It is important that we understand the problem before we begin solving it. Suppose we are given the below problem to solve.

Problem: Write a function which takes two numbers and returns their sum.

To understand this problem there are 5 question we should ask ourselves:

  1. Can I restate the problem in my own words?
    For example, in the above we could say: "add two numbers"
  2. What are the inputs that go into the problem?
    - Are they ints? floats? string for large numbers?
  3. What are the outputs that should come from the solution to the problem?
    - Also are they ints? floats? string?
  4. Can the outputs be determined from the inputs? In other words, do I have enough information to solve the problem?
    - What if one of the inputs is not passed or null, what happens then?
  5. How should I label the important pieces of data that are a part of the problem?
    - What operators, methods, loops are essentials to solve the problem?
Explore concrete examples

Examples provide sanity checks that your eventual solution works how it should. Consider the problem below.

Problem: Write a function which takes in a string and returns counts of each character in the string.

After going through the steps in the above section to understand the problem, lets go through some examples to further understand the above problem:

  1. Start with Simple Examples
    At this point we have determined that the output is an object. Lets start with a simple example like:
    charCount("aaaa"); // output would be {a: 4}
    charCount("hello"); // output would be {h: 1, e: 1, l: 2, o: 1}
  2. Progress to More Complex Examples
    If our input is a sentence do we consider the spaces(or even integers and symbols) as characters? e.g. charCount("my phone number is 1232424");
    Do we also consider case sensitivity? e.g. charCount("Hello hi");
  3. Explore Examples with Empty Inputs
    What do we return when the input is empty: charCount("") // null/undefined/empty object
  4. Explore Examples with Invalid Inputs
    What about if the input is anything other than a string e.g. charCount(123232)
Break it down

Here, you explicitly write out the steps you need to take. This forces you to think about the code you'll write before you write it, and helps you catch any lingering conceptual issues or misunderstandings before you dive in and have to worry about details. Consider the same problem as the above section.

Problem: Write a function which takes in a string and returns counts of each character in the string.

Assuming the instructions are to consider alphanumerics(numbers and letters), ignore spaces and convert uppercase to lowercase letters. We would visualize that the outcome will look something like:
charCount("Your PIN number is 1234!") // { 1: 1, 2: 1, 3: 1, 4: 1, o: 1, u: 2, ... }

So to break it down we would do something like: 

  1. const charCount = (string) => {
  2.     // make object to return at end
  3.  
  4.     // loop over string
  5.       // if the char is a number/letter AND is a key in object, add one to count
  6.       // if the char is a number/letter AND not in object, add it to object and set value to 1
  7.       // if the char is something else(space, period, e.t.c.) don't do anything
  8.  
  9.    // return an object with keys that are lowercase alphanumeric characters
  10. };
Solve or Simplify

Here, you should try to solve the problem if not simplify it. To simplify a problem we should:

  1. Find the core difficulty in what you're trying to do
  2. Temporarily ignore that difficulty
  3. Write a simplified solution
  4. Then incorporate that difficulty back in
Problem: Write a function which takes in a string and returns counts of each character in the string.

Assuming the instructions are to consider alphanumerics(numbers and letters), ignore spaces and convert uppercase to lowercase letters. We would visualize that the outcome will look something like:
charCount("Your PIN number is 1234!") // { 1: 1, 2: 1, 3: 1, 4: 1, o: 1, u: 2, ... }

Lets simplify it as we solve and follow the steps above: 

  1. const charCount = (string) => {
  2.   // make object to return at end
  3.   const result = {};
  4.  
  5.   // convert string to lowercase
  6.   string = string.toLowerCase();
  7.  
  8.   // loop over string
  9.   for(char of string) {
  10. 	if(result[char] > 0) {
  11. 	  // if the char is a number/letter AND is a key in object, add one to count
  12. 	  result[char]++;
  13. 	} else {
  14. 	  // if the char is a number/letter AND not in object, add it to object and set value to 1
  15. 	  result[char] = 1;
  16. 	}
  17.   }
  18.  
  19.  
  20.   // if the char is something else(space, period, e.t.c.) don't do anything (DIFFICULT PART)
  21.  
  22.   // return an object with keys that are lowercase alphanumeric characters
  23.   return result;
  24. };

Now if we incorporate back the difficult parts we get something like:

  1. const charCount = (string) => {
  2.   // make object to return at end
  3.   const result = {};
  4.  
  5.   // convert string to lowercase
  6.   string = string.toLowerCase();
  7.  
  8.   // loop over string
  9.   for(char of string) {
  10. 	// if the char is something else(space, period, e.t.c.) don't do anything (DIFFICULT PART INCORPORATE BACK)
  11. 	/**
  12.         * Below is a regex prototype test() which checks if the param passed matches the regural expression
  13.         * More info: https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/RegExp/test
  14.         */
  15. 	if(/[a-z0-9]/.test(char)) {
  16. 	  if(result[char] > 0) {
  17. 		// if the char is a number/letter AND is a key in object, add one to count
  18. 		result[char]++;
  19. 	  } else {
  20. 		// if the char is a number/letter AND not in object, add it to object and set value to 1
  21. 		result[char] = 1;
  22. 	  }
  23. 	}
  24.   }
  25.  
  26.   // return an object with keys that are lowercase alphanumeric characters
  27.   return result;
  28. };
Look Back and Refactor

Once we solve a problem, it is desirable that we look back at our solution and think of how we can refactor it. This helps us improve on our solution and possibly look at other ways we could solve our problem. There are a couple of questions we can ask ourselves when refactoring our code:

  1. Can you check the result?
  2. Can you derive the result differently?
  3. Can you use the result or method for some other problem?
  4. Can you improve the performance of your solution?
  5. Can you think of other ways to refactor?
  6. How have other people solved this problem?
Problem: Write a function which takes in a string and returns counts of each character in the string.

Assuming the instructions are to consider alphanumerics(numbers and letters), ignore spaces and convert uppercase to lowercase letters. We would visualize that the outcome will look something like:
charCount("Your PIN number is 1234!") // { 1: 1, 2: 1, 3: 1, 4: 1, o: 1, u: 2, ... }

Lets refactor the solution we got in the above section:

  1. const charCount = (string) => {
  2.   // make object to return at end
  3.   const result = {};
  4.  
  5.   // convert string to lowercase
  6.   string = string.toLowerCase();
  7.  
  8.   // loop over string
  9.   for(let char of string) {
  10. 	// if the char is something else(space, period, e.t.c.) don't do anything (DIFFICULT PART INCORPORATE BACK)
  11. 	/**
  12.            * Here, we have refactored to a function that checks the character code of each character and verifies
  13.            * if it is alphanumeric. We do this because regular expressions tend to be slower in execution
  14.           */
  15. 	if(isAlphaNumeric(char)) {
  16. 	  // Refactored from if...else statement
  17. 	  result[char] = ++result[char] || 1;
  18. 	}
  19.   }
  20.  
  21.   // return an object with keys that are lowercase alphanumeric characters
  22.   return result;
  23. };
  24.  
  25. const isAlphaNumeric = (char) => {
  26.   let code = char.charCodeAt(0);
  27.  
  28.   if (!(code > 47 && code < 58) && // numeric (0-9)
  29. 	  !(code > 64 && code < 91) && // upper alpha (A-Z)
  30. 	  !(code > 96 && code < 123)) { // lower alpha (a-z)
  31. 	return false;
  32.   } else {
  33. 	return true;
  34.   }
  35. }
