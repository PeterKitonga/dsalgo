Performance of Arrays & Objects

Big O of Arrays and Objects

With Big O we can analyse how arrays and objects perform based on the operations performed on them using their built-in methods. Arrays and Objects are built-in data types in Javascript that are considered to be Reference data types. This simply means that when you create an object/array and assign it to a variable, the reference to that value is what is set and not the value itself. Our objectives of Big O here is to:

  1. Understand how objects and arrays work, through the lens of Big O
  2. Explain why adding elements to the beginning of an array is costly
  3. Compare and contrast the runtime for arrays and objects, as well as built-in methods

Analyzing Objects

An object is a collection of properties, and a property is an association between a name (or key) and a value. Simply put an object is an unordered list of key value pairs. Consider the object below called instructor.

  1. let instructor = {
  2.     firstName: "Kelly",
  3.     isInstructor: true,
  4.     favoriteNumbers: [1,2,3,4]
  5. };

With the above, we would say that the Big O of Objects is:

  1. Insertion has a Big O of O(1)
  2. Accessing a value has a Big O of O(1)
  3. Removal of a value has a Big O of O(1)
  4. Searching a value has a Big O of O(n)

This means that objects in general are very fast and the only thing that would take a while to perform is searching through an object. Another thing to consider is the Big O of object methods where:

  1. Object.keys() has a Big O of O(n)
  2. Object.values() has a Big O of O(n)
  3. Object.entries() has a Big O of O(n)
  4. {object}.hasOwnProperty() has a Big O of O(1)
Analyzing Arrays

Unlike objects arrays are a ordered list of values and can contain objects even other arrays. Consider the arrays below called names and values.

  1. let names = ["Michael", "Melissa", "Andrea"];
  2.  
  3. let values = [true, {}, [], 2, "awesome"];

Because of the ordered nature that arrays have some operations may take longer than others. With that in mind we see that arrays have a Big O of:

  1. Insertion Big O depends on the methods used to insert into an array. For example, because all array values have an index, if you insert a value at the beginning of the array you will have to reindex all values. This will continue everytime you insert at the beginning of the array O(n). If inserting at the end of the array, this will be faster with a Big O of O(1)
  2. Removal also behaves the same way as insertion where if removing values at the beginning of the array, you will have to reindex the values O(n). Removing values at the end of the array is faster with a Big O of O(1).
  3. Access has a Big O of O(1)
  4. Searching has a Big O of O(n)
