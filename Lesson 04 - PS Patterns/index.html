Problem Solving Patterns

Problem Solving Patterns

Having had a look at the various approaches we take to solve a problem, we will come across various patterns that we use to solve most problems. Some of these patterns include:

  1. Frequency Counter
  2. Multiple Pointers
  3. Sliding Window
  4. Divide and Conquer
  5. Dynamic Programming
  6. Greedy Algorithms
  7. Backtracking
Let's look at some of these below.

Frequency Counter Pattern

This pattern uses objects or sets to collect values/frequencies of values. This can often avoid the need for nested loops or O(N^2) operations with arrays / strings

Problem: Write a function called 'same', which accepts two arrays. The function should return true if every value in the array has it's corresponding value squared in the second array. The frequency of values must be the same.

Simply put our solution will look like:
same([1, 2, 3, 2, 5], [4, 1, 4, 9, 11]) // false because square of 5(25) is not present in the second array
same([1, 2, 3, 2, 5], [4, 1, 4, 9]) // false because length of the second array is not same as the first
same([1, 2, 3, 2, 5], [4, 1, 1, 9, 25]) // false because the frequency of 2 and 1 squared doesn't match in the second array
same([1, 2, 3], [4, 1, 9]) // true because squared values are present and the frequencies match

To solve it, we have two solutions below.

Solution one with Big O of O(n^2). This solution is undesirable because of the use of indexOf() inside the loop which in itself is a loop. This ends up becoming a loop within a loop and has considerable performance effects.

  1. const same = (array_one, array_two) => {
  2.     // check if the lengths of the array match
  3.     if(array_one.length !== array_two.length) {
  4.         return false;
  5.     }
  6.  
  7.     // loop through each item in array one
  8.     for (let item of array_one) {
  9.         // check if the square of item in array one exists in array two
  10.         let index = array_two.indexOf(item ** 2);
  11.  
  12.         // if not found return false
  13.         if (index === -1) {
  14.             return false;
  15.         }
  16.  
  17.         // remove the first item found and move to the next
  18.         array_two.splice(index, 1);
  19.     }
  20.  
  21.     return true;
  22. };

Solution two with Big O of O(n). This solution is the most optimal and will run faster and better than the above. It does have three loops, however, they are seperate and will loop less times than the above solution.

  1. const same = (array_one, array_two) => {
  2.     // check if the lengths of the array match
  3.     if(array_one.length !== array_two.length) {
  4.         return false;
  5.     }
  6.  
  7.     // initialize empty objects that will house the frequency of each array item
  8.     let frequency_counter_one = {};
  9.     let frequency_counter_two = {};
  10.  
  11.     // populates the first frequency counter
  12.     for (let item_one of array_one) {
  13.         frequency_counter_one[item_one] = ++frequency_counter_one[item_one] || 1;
  14.     }
  15.  
  16.     // populates the second frequency counter
  17.     for (let item_two of array_two) {
  18.         frequency_counter_two[item_two] = ++frequency_counter_two[item_two] || 1;
  19.     }
  20.  
  21.     // loop and check frequency of the squared value in array two
  22.     for (let number in frequency_counter_one) {
  23.         // check if the square of the value exists in the second frequency counter
  24.         if(!(number ** 2 in frequency_counter_two)) {
  25.             return false;
  26.         }
  27.  
  28.         // check if frequency of the squared value matches
  29.         if (frequency_counter_one[number] !== frequency_counter_two[number ** 2]) {
  30.             return false;
  31.         }
  32.     }
  33.  
  34.     return true;
  35. };
Problem: Given two strings, write a function to determine if the second string is an anagram of the first. An anagram is a word, phrase, or name formed by rearranging the letters of another, such as cinema, formed from iceman.

To visualize it, the function will look like: validAnagram('texttwisttime', 'timetwisttext') // true 

  1. const validAnagram = (string_one, string_two) => {
  2.     // check if the length of the strings match
  3.     if (string_one.length !== string_two.length) {
  4.         return false;
  5.     }
  6.  
  7.     // create objects with frequency for each character
  8.     let char_frequency_one = {};
  9.     let char_frequency_two = {};
  10.  
  11.     for (let char_one of string_one) {
  12.         char_frequency_one[char_one] = ++char_frequency_one[char_one] || 1;
  13.     }
  14.  
  15.     for (let char_two of string_two) {
  16.         char_frequency_two[char_two] = ++char_frequency_two[char_two] || 1;
  17.     }
  18.  
  19.     // loop through each character
  20.     for (let character in char_frequency_one) {
  21.         // check if character in first object exists in second
  22.         if (!char_frequency_two.hasOwnProperty(character)) {
  23.             return false;
  24.         }
  25.  
  26.         // check if frequency of character matches in the second object
  27.         if (char_frequency_two[character] !== char_frequency_one[character]) {
  28.             return false;
  29.         }
  30.     }
  31.  
  32.     return true;
  33. }
Problem: Given two strings, a and b, that may or may not be of the same length, determine the minimum number of character deletions required to make a and b anagrams. Any characters can be deleted from either of the strings.

To visualize it, the function will look like: makeAnagram('accfgilmmnrsvwxxyyyz', 'bbdeeghhijjklmmmooppqrrstuvwwx') // 30 

  1. const makeAnagram = (a, b) => {
  2.     // create frequency counter objects for each character
  3.     let frequency_counter_one = {};
  4.     let frequency_counter_two = {};
  5.  
  6.     for (let char_one of a) {
  7.         frequency_counter_one[char_one] = ++frequency_counter_one[char_one] || 1
  8.     }
  9.  
  10.     for (let char_two of b) {
  11.         frequency_counter_two[char_two] = ++frequency_counter_two[char_two] || 1
  12.     }
  13.  
  14.     // initialize counter for deletions
  15.     let deletions = 0;
  16.  
  17.     // Loop through the first object
  18.     for (let character in frequency_counter_one) {
  19.         // check if char doesn't exist in second object, increment deletions counter
  20.         if (!frequency_counter_two.hasOwnProperty(character)) {
  21.             deletions += frequency_counter_one[character];
  22.         } else {
  23.             // if character frequency does not match, subtract both frequencies to get deletions
  24.             if (frequency_counter_one[character] !== frequency_counter_two[character]) {
  25.                 deletions += Math.abs(frequency_counter_one[character] - frequency_counter_two[character]);
  26.             }
  27.         }
  28.     }
  29.  
  30.     // Loop through the second object
  31.     for (let character in frequency_counter_two) {
  32.         // check if char doesn't exist in second object, increment deletions counter
  33.         if (!frequency_counter_one.hasOwnProperty(character)) {
  34.             deletions += frequency_counter_two[character];
  35.         }
  36.     }
  37.  
  38.     return deletions;
  39. }
Multiple Pointers Pattern

This pattern involves creating pointers or values that correspond to an index or position and moving towards the beginning, end or middle based on a certain condition. It is very efficient for solving problems with minimal space complexity as well.

Problem: Write a function called sumZero which accepts a sorted array of integers. The function should find the first pair where the sum is 0. Return an array that includes both values that sum to zero or undefined if a pair does not exist.

To visualize it, the function will look like: sumZero([-3,-2,-1,0,1,2,3]) // [-3,3] because this pair is the first to give a sum of zero

Solution one with a big O of O(n^2)

  1. const sumZero = (arr) => {
  2.     // loop through each item in the array
  3.     for (let i = 0; i < arr.length; i++) {
  4.         // get the current value and loop through the rest while testing condition
  5.         for (let j = i + 1; j < arr.length; j++) {
  6.             // if current value i plus current value j equals 0, return pair
  7.             if (arr[i] + arr[j] === 0) {
  8.                 return [arr[i], arr[j]];
  9.             }
  10.         }
  11.     }
  12. }

Refactored solution two with a big O of O(n)

  1. const sumZero = (arr) => {
  2.     // initialize the left and right indices
  3.     let left = 0;
  4.     let right = arr.length - 1;
  5.  
  6.     // loop through each item in the array while the left index is less than the right
  7.     while (left < right) {
  8.         /**
  9.          * Get the current left value and sum the current right value. 
  10.          * If sum equals 0, return pair
  11.         */
  12.         let sum = arr[left] + arr[right];
  13.         if (sum === 0) {
  14.             return [arr[left], arr[right]]
  15.         } else if (sum > 0) {
  16.             // decrement right index if sum is greater than 1
  17.             right--;
  18.         } else {
  19.             // increment left index if sum is less than 1
  20.             left++;
  21.         }
  22.     }        
  23. }
Problem: Implement a function called countUniqueValues, which accepts a sorted array, and counts the unique values in the array. There can be negative numbers in the array, but it will always be sorted.

To visualize it, the function will look like: countUniqueValues([1,2,3,4,4,4,7,7,12,12,13]) // 7

Solution one with a big O of O(n^2). Note: this does not use multiple pointer pattern

  1. const countUniqueValues = (arr) => {
  2.     // check length of the array, return 0 if empty
  3.     if (arr.length === 0) {
  4.         return 0;
  5.     }
  6.  
  7.     // initialize array for unique values
  8.     const unique = [];
  9.  
  10.     // loop through each item in array
  11.     for (let index = 0; index < arr.length; index++) {
  12.         // push values if not in array
  13.         if (!unique.includes(arr[index])) {
  14.             unique.push(arr[index])
  15.         }
  16.     }
  17.  
  18.     return unique.length;
  19. }

Refactored solution two with a big O of O(n)

  1. const countUniqueValues = (arr) => {
  2.     // check length of the array, return 0 if empty
  3.     if (arr.length === 0) {
  4.         return 0;
  5.     }
  6.  
  7.     // initialize a counter
  8.     let counter = 0;
  9.  
  10.     // loop through each value starting from index 1
  11.     for (let index = 1; index < arr.length; index++) {
  12.         // compare value at index with at counter, if not same increment counter and update arr at index counter
  13.         if (arr[counter] !== arr[index]) {
  14.             counter++;
  15.             arr[counter] = arr[index];
  16.         }
  17.     }
  18.  
  19.     // return counter plus 1 
  20.     return counter + 1;
  21. }
Sliding Window Pattern

This pattern involves creating a window which can either be an array or number from one position to another. Depending on a certain condition, the window either increases or closes (and a new window is created). It is very useful for keeping track of a subset of data in an array/string.

Problem: Write a function called maxSubarraySum which accepts an array of integers and a number called n. The function should calculate the maximum sum of n consecutive elements in the array.

To visualize it, the function will look like: maxSubarraySum([1,2,5,2,8,1,5], 4) // 17

In this solution we have a big O of O(n^2).

  1. const maxSubarraySum = (arr, num) => {
  2.     // check the length of the array
  3.     if (num > arr.length) {
  4.         return null;
  5.     }
  6.  
  7.     // initialize a max store
  8.     let max = -1;
  9.  
  10.     // loop through values to length minus num plus one
  11.     for (let i = 0; i < arr.length - (num + 1); i++) {
  12.         // initialize a temp sum for the max store of each set
  13.         let temp = 0;
  14.  
  15.         // loop to the numth value from current index
  16.         for (let j = 0; j < num; j++) {
  17.             // increment the temp sum for this set
  18.             temp += arr[i + j];
  19.  
  20.             // check if the temp sum is greater than max store, equate max to temp if greater
  21.             if(temp > max) {
  22.                 max = temp;
  23.             }
  24.         }
  25.     }
  26.  
  27.     // return max
  28.     return max;
  29. }

Lets refactor it to have a big O of O(n). Like so:

  1. const maxSubarraySum = (arr, num) => {
  2.     // check length of array
  3.     if (num > arr.length) {
  4.         return null;
  5.     }
  6.  
  7.     // initialize the temp and max store
  8.     let temp = 0;
  9.     let max = 0;
  10.  
  11.     // add the first numth values and set max store
  12.     for (let i = 0; i < num; i++) {
  13.         max += arr[i];
  14.     }
  15.  
  16.     // set temp store same as max 
  17.     temp = max;
  18.  
  19.     // loop through values starting from numth index
  20.     for (let i = num; i < arr.length; i++) {
  21.         // update temp store by removing the (index - num) value and adding the current index value
  22.         temp = temp - arr[i - num] + arr[i];
  23.         // if temp store is greater than max, update max
  24.         if (temp > max) {
  25.             max = temp;
  26.         }
  27.     }
  28.  
  29.     // return max
  30.     return max;
  31. }
Divide and Conquer Pattern

This pattern involves dividing a data set into smaller chunks and then repeating a process with a subset of data. This pattern can tremendously decrease time complexity.

Problem: Given a sorted array of integers, write a function called search, that accepts a value and returns the index where the value passed to the function is located. If the value is not found, return -1.

To visualize it, the function will look like: search([1,2,3,4,5,6], 6) // 5

The simple and naive solution would be to do as below. This solution has a time complexity of O(n). 

  1. const search = (arr, num) => {
  2.     // initialize result to -1, this will be the result if value is not found
  3.     let result = -1;
  4.  
  5.     // loop through each value and check if equates to num
  6.     for (let i = 0; i < arr.length; i++) {
  7.         // if num is found, return index
  8.         if (arr[i] === num) {
  9.             return i;
  10.         }
  11.     }
  12.  
  13.     return result;
  14. }

Using divide and conquer we can as below. This introduces binary search and it has a time complexity of O(log n)

  1. const search = (arr, num) => {
  2.     // initialize min and max values for our iteration
  3.     let min = 0;
  4.     let max = arr.length - 1;
  5.  
  6.     // iterate within the bounds of min and max
  7.     while(min <= max) {
  8.         // set the middle of the array
  9.         let middle = Math.floor((max + min) / 2);
  10.  
  11.         // check if the middle value is greater than the num
  12.         if (arr[middle] > num) {
  13.             // if greater than num, decrement middle by one and set to max
  14.             max = middle - 1;
  15.         } else if (arr[middle] < num) {
  16.             // if less than num, increment min by one
  17.             min = middle + 1
  18.         } else {
  19.             // if equal to num, return middle
  20.             return middle;
  21.         }
  22.     }
  23.  
  24.     // return -1 if not found
  25.     return -1;
  26. }
